weather-mcp Architecture

An AgentBack MCP server over Open-Meteo · five layers, dependencies flow strictly downward

One wiring, three transports

  • • main.ts — stdio for Claude Desktop / Cursor
  • • serve-http.ts — Streamable HTTP at /mcp
  • • console.ts — dev inspector UI
  • • All call registerWeatherMcp() — stay in lockstep

Contracts as single source of truth

  • • Each Zod schema = validator + z.infer type + MCP schema
  • • keys.ts ties DI BindingKeys to their types
  • • Both adapter & domain depend only downward on these
  • • keys ↔ service edge is type-only (erased at runtime)

Domain & the one missing seam

  • • WeatherService — stateless @injectable singleton
  • • Tools delegate to it via @inject(WEATHER_SERVICE)
  • • Coarse seam: override the whole binding in tests
  • • Gap: global fetch has no fine-grained test seam
